# Product Documentation Generation Prompt
Project: Privantix Source Inspector

This document defines the instructions for generating the official documentation of the product.

The AI must treat this document as the authoritative specification for creating and organizing the documentation of the project.

---

# 1 Purpose

Generate a complete and structured documentation set for the product **Privantix Source Inspector**, based on the current repository implementation.

The documentation must explain:

- what the product does
- how to install and run it
- how the architecture works
- what outputs it produces
- how users should interpret the results

The documentation must be clear for both technical and semi-technical users.

---

# 2 Product Context

Privantix Source Inspector is a portable data repository analyzer that scans folders and identifies structural characteristics of tabular datasets.

The tool analyzes files such as:

- CSV
- TXT
- TSV
- XLSX

The tool extracts technical metadata including:

- encoding
- delimiter
- header detection
- column count
- inferred column types
- null ratios
- sample values

The results are exported as structured outputs to help organizations understand the structure of their data repositories.

The generated documentation must:
