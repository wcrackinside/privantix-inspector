# Product Website Generation Prompt
Project: Privantix Source Inspector

This document defines the instructions for generating the official product website content.

The AI must treat this document as the authoritative specification for creating the web presence of the product.

The website must reflect the real capabilities of the product and must not invent features that are not implemented.

---

# 1 Purpose

Generate the full website content for **Privantix Source Inspector**.

The website must:

- clearly explain the product
- communicate the problem it solves
- describe its capabilities
- provide download instructions
- link to documentation
- present the tool as a professional software product

The website must be structured so it can be easily deployed as a static site.

---

# 2 Product Context

Privantix Source Inspector is a **portable repository analysis tool** that scans directories and analyzes tabular data files.

The tool inspects datasets and extracts technical metadata including:

- encoding detection
- delimiter detection
- header detection
- column structure
- inferred data types
- null ratios
- sample values

The goal is to help organizations understand the structure of their data repositories.

The tool is portable and requires no infrastructure.

---

# 3 Target Audience

The website should be written for:

- data engineers

- data governance teams
- technical auditors
- organizations evaluating their data repositories

The language should be professional but accessible.
