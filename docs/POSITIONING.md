# Positioning

## Positioning Statement

Privantix provides lightweight repository intelligence and governance tooling for organizations that need visibility into their data without adopting large enterprise platforms.

## Core Value Proposition

Privantix helps organizations answer practical questions such as:

- What datasets do we have?
- How are they structured?
- What sensitive patterns are present?
- Who can access these repositories?
- Which files or folders are potentially overexposed?

## Market Position

Privantix is not intended to replace large enterprise governance suites.

Instead, it is positioned as:

- a practical starting point for data governance
- a portable inspection toolkit
- a lightweight bridge between unmanaged repositories and mature governance practices

## Comparison to the Market

### Compared with enterprise governance platforms

Platforms such as Collibra, BigID or OneTrust offer broad enterprise workflows, but they typically require more budget, more implementation effort and more organizational maturity.

Privantix focuses on:

- portability
- fast time to value
- repository-level inspection
- practical metadata extraction
- lightweight governance adoption

### Compared with technical metadata tools

Open-source metadata tools often focus on data engineering environments, catalogs and platform integration.

Privantix focuses more directly on:

- filesystem repositories
- tabular file inspection
- structural profiling
- ACL audit
- operational repository visibility

## Ideal Audience

Privantix is designed for:

- data teams auditing repositories
- governance teams building foundational controls
- technical auditors
- organizations with legacy shared folders or unmanaged data storage
- teams that need an easy entry point into governance practices

## Product Narrative

Privantix helps organizations move from:

- unknown repositories
- undocumented datasets
- unclear access exposure

toward:

- structured technical visibility
- inspection-based governance
- practical repository control

## Differentiators

- Portable execution
- Focus on real repository problems
- Support for multiple file formats and archives
- Governance-aware features such as sample hiding, checksums and ACL inspection
- Modular product line instead of one monolithic platform
